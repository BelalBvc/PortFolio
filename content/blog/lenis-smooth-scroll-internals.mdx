Native CSS `scroll-behavior: smooth` is janky and uncontrollable. Lenis solves this with lerp interpolation on a single RAF loop.

## The Core Loop

```ts
const lenis = new Lenis({ lerp: 0.1 })
function raf(time: number) {
  lenis.raf(time)
  requestAnimationFrame(raf)
}
requestAnimationFrame(raf)
```

<Callout type="info">
  `lerp` (linear interpolation) eases the scroll position toward the target each frame. Lower = smoother but laggier. 0.1 is the sweet spot.
</Callout>

## Why Not CSS?

CSS smooth-scroll can't be scrubbed, can't be tied to animation libraries, and behaves differently across browsers. Lenis gives you a single scroll value to drive everything.

```ts
lenis.on('scroll', (e) => {
  ScrollTrigger.update() // GSAP stays in sync
})
```

## Integration with GSAP

The key insight: hand the RAF loop to GSAP's ticker so there's only one loop running.

```ts
gsap.ticker.add((t) => lenis.raf(t * 1000))
gsap.ticker.lagSmoothing(0)
```

This is why the portfolio feels buttery — one loop, perfectly synced.
